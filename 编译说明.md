# 物理實驗報告自動化系統 - C 語言版

## 項目簡介
這是一個完整的 C 語言實現的物理實驗報告自動化系統，具有以下功能：

✓ 命令行交互式輸入（日期、名稱、儀器、內容）
✓ 通過 NVIDIA NIM API 進行 AI 智能潤色
✓ 自動生成 WORD (.docx) 和 PDF 文件
✓ 支持多組員心得自動整理
✓ 完整的錯誤處理和重試機制

---

## 開發環境要求

### Windows 系統

1. **MinGW-w64 (C 編譯器)**
   - 下載：https://www.mingw-w64.org/ 或 https://github.com/niXman/mingw-builds-binaries/releases
   - 或使用便攜版本 (推薦)：https://nuwen.net/mingw.html

2. **必要的庫**
   - libcurl (用於 HTTP 請求)
   - libcjson (用於 JSON 解析)
   - OpenSSL (HTTPS 支持)

3. **Python 3.x**
   - 用於生成 .docx 和 .pdf 文件
   - 已包含的 Python 庫：
     - python-docx
     - docxtpl
     - docx2pdf

## 完整安裝步驟

### 方式 1：使用預編譯的二進制文件（推薦新手）

#### 第 1 步：安裝 MinGW-w64 便攜版

1. 訪問 https://nuwen.net/mingw.html
2. 下載最新版本 (如 `mingw-14.1-without-git.zip`)
3. 解壓到 `C:\mingw` 或任意位置
4. 將解壓路徑下的 `\bin` 目錄加入 Windows 環境變數：
   - 右鍵「此電腦」→ 屬性 → 進階系統設定 → 環境變數
   - 在「系統變數」中新增/編輯 PATH，添加解壓目錄的 bin 路徑
   - 例如：`C:\mingw\bin`

#### 第 2 步：安裝必要的庫

打開 PowerShell 或 cmd，執行：

```bash
# 安裝 libcurl 和 cJSON (使用 vcpkg 或預編譯包)
# 方式 A：使用 vcpkg (如果已安裝)
vcpkg install curl:x64-windows cjson:x64-windows

# 方式 B：手動下載預編譯版本
# 從以下地址下載並放入 C:\mingw\
# - https://curl.se/download/
# - https://github.com/DaveGamble/cJSON/releases

# 方式 C：使用 MSYS2 包管理器（推薦）
pacman -S mingw-w64-x86_64-curl mingw-w64-x86_64-cjson mingw-w64-x86_64-openssl
```

#### 第 3 步：驗證安裝

```bash
gcc --version
curl --config
python --version
```

#### 第 4 步：編譯程序

```bash
cd C:\Users\wangb\OneDrive\Desktop\test word C\WORD_CLI
make all
```

成功後會生成 `lab_report_generator.exe`

## 使用步驟

### 第 1 步：準備環境

確保以下文件在 WORD_CLI 目錄中：
- `main.c` - 主程序源碼
- `template.docx` - WORD 範本（已自動複製）
- `Makefile` - 編譯配置

### 第 2 步：編譯

```bash
# 導航到程序目錄
cd "C:\Users\wangb\OneDrive\Desktop\test word C\WORD_CLI"

# 執行編譯
make all

# 如果遇到錯誤，嘗試清理後重新編譯
make clean
make all
```

### 第 3 步：運行程序

```bash
# 方式 A：直接執行
./lab_report_generator.exe

# 方式 B：使用 make
make run

# 方式 C：雙擊 lab_report_generator.exe（Windows 圖形界面）
```

### 第 4 步：交互式輸入

程序運行後會依次要求輸入：

1. **實驗日期**（默認：2026/4/8）
2. **實驗名稱**（默認：實驗4 電力線之分布）
3. **材料與儀器清單**
   - 輸入儀器名稱
   - 輸入數量
   - 輸入備註
   - 名稱留空時結束輸入
4. **實驗內容草稿**
   - 實驗目的
   - 實驗原理
   - 步驟/方法
   - 結果與討論
5. **三位組員心得**
   - 王靖宏心得
   - 廖承厚心得
   - 陳祈佑心得

### 第 5 步：自動生成

程序會自動：
1. 將所有草稿打包發送給 NVIDIA NIM API
2. 接收 AI 潤色後的內容
3. 使用 Python 腳本將內容填充到 WORD 範本
4. 生成 `實驗報告_成品_穩定版.docx`
5. 嘗試生成 `實驗報告_成品_穩定版.pdf`

## 詳細編譯命令

### 從零開始完整編譯

```bash
# 1. 進入項目目錄
cd "C:\Users\wangb\OneDrive\Desktop\test word C\WORD_CLI"

# 2. 清理舊編譯文件
make clean

# 3. 完整編譯
gcc -Wall -Wextra -std=c99 -o lab_report_generator.exe main.c -lcurl -lcjson

# 或使用 Makefile
make all

# 4. 驗證編譯結果
dir /b lab_report_generator.exe

# 5. 運行測試
./lab_report_generator.exe
```

### 分步編譯（詳細模式）

```bash
# 只編譯，不鏈接
gcc -Wall -Wextra -std=c99 -c main.c -o main.o

# 鏈接生成可執行文件
gcc -o lab_report_generator.exe main.o -lcurl -lcjson

# 查看生成的文件
dir /b *.exe *.o
```

### 編譯故障排除

#### 錯誤：`gcc: command not found`

**解決方案：**
```bash
# 方案 1：檢查 PATH（Windows）
echo %PATH%

# 方案 2：使用完整路徑編譯
"C:\mingw\bin\gcc" -version

# 方案 3：添加到 PATH（PowerShell）
$env:PATH += ";C:\mingw\bin"
```

#### 錯誤：`undefined reference to 'curl_easy_init'`

**解決方案：**
```bash
# 檢查 libcurl 是否安裝
curl-config --libs

# 手動指定庫路徑
gcc -o lab_report_generator.exe main.c -L"C:\mingw\lib" -lcurl -lcjson
```

#### 錯誤：`cjson/cJSON.h: No such file or directory`

**解決方案：**
```bash
# 檢查 cJSON 頭文件位置
find /usr -name "cJSON.h" 2>/dev/null

# 或指定包含路徑
gcc -I"C:\mingw\include" -o lab_report_generator.exe main.c -lcurl -lcjson
```

## 常見問題 (FAQ)

**Q: 編譯後文件很大嗎？**
A: 約 1-5 MB，取決於靜態鏈接的庫。使用 `strip` 可減少大小。

**Q: API 密鑰會被洩露嗎？**
A: 密鑰已嵌入代碼中。生產環境應使用環境變數替代。

**Q: 支持其他操作系統嗎？**
A: 代碼是跨平台的，僅需在 Linux/macOS 上修改編譯配置。

**Q: 能否離線使用？**
A: 不能，需要 NVIDIA API 進行 AI 潤色。
